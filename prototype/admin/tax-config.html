v1 uses manual tax entry

A configurable Indian income tax slab engine is not part of v1 — it is on the v2 roadmap. For now, the PayrollOfficer enters tax manually on each payslip during the payroll run review. The system shows a reference figure derived from a standard formula, but the value is editable per payslip.

Standard Reference Formula

Used to suggest a tax figure when generating each payslip. PayrollOfficer can override per row.

reference_tax = gross_taxable_income × flat_reference_rate
%

Currently 9.5% · Used as a guide only

Definition used to compute the reference figure

How this appears during a payroll run

In the run review screen, each payslip row shows a Tax (reference) column and an editable Tax (final) field that the PayrollOfficer can adjust.

PayrollOfficer reviews each row, adjusts where needed (statutory exemptions, prior reconciliations), then submits the run to Admin for finalisation.

v2 — Indian Tax Slab Engine

Planned for the next release. The engine will replace manual entry with bracket-based computation under the New Regime, with annual slab updates configurable by Admin.

Out of scope for v1: bracketed slab editor, regime selection, surcharge, cess, 80C/80D investments, Form 16 auto-generation. These remain manual in v1 and ship in v2.
